# powledge-toolkit — Agent & Contributor Guidelines

## Project Purpose

AI-powered personal knowledge base. Raw source material is ingested from external systems (Confluence, file drops, web clips), cleaned into structured knowledge files, indexed, and made searchable by agents and humans alike.

---

## Repository Structure

```
powledge.config.json        ← global config (dirs, ingest, etl, claude, confluence)
config.py                   ← config loader (load_config, raw_dir, knowledge_dir, index_dir)
ingest/
  confluence.py             ← Confluence REST API v2 ingestion → raw/
etl/
  build-index.py            ← aggregate index/*.index.json → index/all.json with related links
search/
  list-index.py             ← agent entry point: list/filter/project index entries
raw/                        ← ingested files: <id>.md + <id>.meta.json sidecars
knowledge/                  ← clean knowledge files: <id>.md with YAML frontmatter
index/
  <id>.index.json           ← one per knowledge file (lightweight descriptor)
  all.json                  ← aggregated index (generated by etl/build-index.py)
tests/
  test_confluence.py
  test_list_index.py
  test_build_index.py
```

## Build & Test Commands

```bash
# Run all tests
task test
python -m pytest tests/ -v

# Run a specific test file
python -m pytest tests/test_list_index.py -v
python -m pytest tests/test_build_index.py -v

# Ingest from Confluence (requires env vars — see Confluence section below)
task confluence                    # all configured spaces/pages, then rebuilds index
task confluence:page -- 111 222    # specific page IDs, then rebuilds index
task confluence:space -- ENG HR    # specific space keys, then rebuilds index
task confluence:dry                # preview, no files written

# Build aggregated index
task index:build

# Search / list index entries
task search:index                              # all entries as JSON array
task search:index -- --search "query"          # keyword filter
task search:index -- --tags ai,etl            # tag filter (AND)
task search:index -- --fields id,title,tags   # field projection
task search:index -- --format lines           # newline-delimited JSON
task search:index:tags -- ai etl              # shorthand for tag filter

# Inspect store
task ls:raw
task ls:knowledge
task ls:index
task status

# Initialise directories
task init
```

## Agent Workflow

Agents (LLM-based or scripted) should follow this order when querying the KB:

1. **Discover** — call `python search/list-index.py --fields id,title,summary,tags` to get a compact overview of what is in the KB.
2. **Filter** — use `--tags` or `--search` to narrow to relevant entries.
3. **Fetch** — read `knowledge/<id>.md` directly for the full content of specific entries.
4. **Related** — use the `related` field in `index/all.json` to find adjacent entries worth including as context.

`index/all.json` is the richer structured view; `search/list-index.py` reads individual `*.index.json` files and is always up-to-date even before `index:build` has run.

## Per-File Index Schema (`index/<id>.index.json`)

```json
{
  "id": "abc123",
  "title": "...",
  "summary": "2-3 sentence description",
  "tags": ["ai", "etl"],
  "source": "https://...",
  "date": "2026-05-09",
  "knowledge_file": "knowledge/abc123.md"
}
```

## Aggregated Index Schema (`index/all.json`)

```json
{
  "generated_at": "2026-05-11T10:00:00Z",
  "count": 42,
  "entries": [
    {
      "id": "abc123",
      "title": "...",
      "summary": "...",
      "tags": ["ai"],
      "source": "...",
      "date": "...",
      "knowledge_file": "knowledge/abc123.md",
      "related": ["def456", "ghi789"]
    }
  ]
}
```

`related` — up to 5 IDs that share at least one tag, sorted by shared-tag count descending. Recomputed every time `etl/build-index.py` runs.

## Confluence Ingestion

Requires three environment variables:

```bash
CONFLUENCE_BASE_URL=https://yourco.atlassian.net/wiki
CONFLUENCE_EMAIL=you@example.com
CONFLUENCE_API_TOKEN=<token>
```

Configure spaces and pages in `powledge.config.json` under `confluence.space_keys` / `confluence.page_ids`. Pages labelled `draft` or `archived` are skipped by default (`confluence.exclude_labels`).

Confluence tasks (`task confluence`, `task confluence:page`, `task confluence:space`) automatically run `task index:build` on completion. Dry-run variants do not.

## Coding Conventions

- **stdlib only** — no third-party packages anywhere in this project.
- Scripts use `argparse` for CLI flags.
- All file I/O uses `encoding="utf-8"` explicitly.
- Scripts insert their parent's parent onto `sys.path` so `config.py` is importable regardless of working directory.
- Pure functions (load, filter, transform) are defined at module level and imported directly in tests — avoid burying logic inside `main()`.
- `main()` is only called via `if __name__ == "__main__"`.

## Testing Conventions

- Framework: `unittest` (stdlib), collected by pytest.
- Test files mirror the script they test: `tests/test_<script-stem>.py`.
- Scripts with hyphens in their filename are loaded via `importlib.util.spec_from_file_location`.
- Use `tempfile.TemporaryDirectory` for all file-system tests; never write to `raw/`, `knowledge/`, or `index/` from tests.
- Mock `load_config` and `index_dir` in integration tests that exercise `main()`.
- Cover: happy path, empty input, missing/malformed files, edge cases (single entry, no tags, capped limits).
